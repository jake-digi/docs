## Update Participant

Update an existing participant. Only include fields you want to update.

<ParamField path="id" type="string" required>
  Participant ID
</ParamField>

<ParamField body="first_name" type="string">
  First name
</ParamField>

<ParamField body="surname" type="string">
  Surname
</ParamField>

<ParamField body="gender" type="string">
  Gender
</ParamField>

<ParamField body="status" type="string">
  Status
</ParamField>

<ParamField body="super_group_id" type="string">
  Supergroup ID
</ParamField>

<ParamField body="sub_group_id" type="string">
  Subgroup ID (must belong to supergroup if provided)
</ParamField>

<ParamField body="block_id" type="string">
  Block ID
</ParamField>

<ParamField body="room_id" type="string">
  Room ID (must belong to block if provided)
</ParamField>

<Warning>
  When updating relationships, ensure they are valid:
  - Subgroup must belong to the supergroup (if both provided)
  - Room must belong to the block (if both provided)
</Warning>

### Example Request

```bash
curl -X PUT https://api.checkpoint.b-digital.uk/api/v1/participants/API_TEST_001 \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "first_name": "John",
    "surname": "Smith",
    "rank": "Senior"
  }'
```
